以下の機能仕様に基づいて、このダッシュボードプロジェクトへの新機能を実装してください。

## 機能仕様

$ARGUMENTS

## テンプレート

@templates/feature_template.txt

## 実装手順

1. 上記テンプレートの各 `{{プレースホルダー}}` を機能仕様から読み取った値で置き換える

2. プレースホルダーの対応表：
   | プレースホルダー | 仕様から読み取る値 |
   |----------------|-----------------|
   | `{{FEATURE_NAME}}` | 機能の表示名（日本語可） |
   | `{{DESCRIPTION}}` | 機能の概要説明 |
   | `{{DATA_SOURCE_URL}}` | データ取得先の URL |
   | `{{REVALIDATE_SECONDS}}` | キャッシュ有効期間（秒）|
   | `{{CACHE_TAG}}` | キャッシュタグ名（kebab-case） |
   | `{{TypeName}}` | TypeScript 型名（PascalCase） |
   | `{{FeatureName}}` | 機能名（PascalCase、コンポーネント名に使用） |
   | `{{camelName}}` | 機能名（camelCase、変数名に使用） |
   | `{{type_file}}` | 型定義ファイル名（拡張子なし） |
   | `{{fetcher_file}}` | フェッチャーファイル名（拡張子なし） |
   | `{{fields}}` | TypeScript インターフェースのフィールド定義 |

3. STEP 1〜5 の順にファイルを実際に作成・編集する

4. 仕様に不明点があれば合理的な値を仮定し、コメントで明記する

5. 実装完了後、テンプレートの「実装後チェックリスト」を確認して報告する
