現在の Git ブランチ状態を調査し、以下の手順でレポートを生成してください。

## 手順

以下のコマンドを順番に実行して情報を収集してください：

1. `git branch --show-current` — 現在のブランチ名を取得
2. `git log -n 5 --oneline` — 直近5件のコミット履歴を取得
3. `git status --short` — 変更ファイルの一覧を取得
4. `git diff --stat HEAD` — 変更のあったファイルと行数を取得
5. `git log --oneline main..HEAD 2>/dev/null || git log --oneline master..HEAD 2>/dev/null` — デフォルトブランチとの差分コミット数を取得

## 出力フォーマット

収集した情報を以下の Markdown 形式でレポートとして出力してください：

```
# ブランチ状態レポート

生成日時: YYYY-MM-DD HH:MM

## 基本情報
- **現在のブランチ**: <ブランチ名>
- **デフォルトブランチとの差分**: <N> コミット先行

## 直近のコミット履歴（最新5件）
| ハッシュ | メッセージ |
|----------|-----------|
| abc1234  | コミットメッセージ |
...

## 変更ファイル一覧
| 状態 | ファイルパス |
|------|-------------|
| M    | src/xxx.ts  |
...

## 変更サマリー
- 変更ファイル数: N 件
- 追加行数: +N
- 削除行数: -N

## 所感
変更内容から読み取れる作業の目的や注意点を2〜3文で簡潔に記述してください。
```

変更ファイルがない場合や、デフォルトブランチと同一の場合もその旨を明記してください。
